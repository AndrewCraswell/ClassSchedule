Bellevue College instructions for Upgrading the Class Schedule
             application from ver 1 to ver 3.

1.  Rename the 'app_offline.html' file at the following locations to 'app_offline.htm'

    /mnt/auto/bacon/www.bellevuecollege.edu/classes
    /mnt/auto/ham/www.bellevuecollege.edu/classes

*******************************************
*    THE CLASS SCHEDULE IS NOW OFFLINE    *
*******************************************

(NOTE: The script in step #2 will start by backing up the Class Schedule database.)

2.  Run the SQL script:  '1-ClassSchedule.DB-Upgrade-v1.8-v3.sql'

3.  Deploy the ver 3 Class Schedule application.

4.  Rename the 'app_offline.htm' file at the following locations to 'app_offline.html'

    /mnt/auto/bacon/www.bellevuecollege.edu/classes
    /mnt/auto/ham/www.bellevuecollege.edu/classes

*******************************************
*  THE CLASS SCHEDULE IS NOW BACK ONLINE  *
*******************************************

5.  PERFORM A SANITY CHECK - make sure the updated application runs without errors.

6.  Run the SQL script:  '2-ClassSchedule.DB-PostUpgradeCleanUp-v1.8-v3.sql'

7.  PERFORM *ANOTHER* SANITY CHECK - make sure the updated application runs without errors.

8.  Update the CanvasExtensions database:

    1.  Replace the ClassData synonym with one (of the same name) that
        references ClassSchedule..SectionsMeta (instead of SectionFootnotes).

    2.  ALTER the vw_Class view: Rename the 'CustomTitle' column of the ClassData
        synonym to 'Title'
